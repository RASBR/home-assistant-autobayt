# Devices

## Device Model Naming Convention

Example `SW100PMV1` that can represent the following:
  * **SW** First two characters represent device Type/Category; in this case **SWITCH**.
  * **100** Characters 3-5 represent device Model; in this case **SINGLE** button switch.
  * **PM** Characters 6 or 6-7 represent device Features; in this case **POWER METER**
  * **V1** The Letter `V` means device *Version* and the number after `V` is device *Version*; in this case **1**.

## Device ID vs. MAC Address
Device ID is the device MAC address without the "colon" `:` symbols.

## Devices

### SWITCHES - code `SW`

1. **SW100HV1:** One-button switch version 1.
2. **SW100PMV1:** One-button switch version 1 with Power Metering.
3. **SW200LV1:** Two-button switch version 1.
4. **SW300LV1:** Three-button switch version 1.

#### Models
* **100** *One button*
* **200** *Two buttons*
* **300** *Three buttons*

#### Features
* **H** `*Meaning*`
* **L** `*Meaning*`
* **PM** *Power Metering*
